# 워크스페이스 운영 가이드

이 PC는 **실행·개발 런타임** 역할만 하고, 소스·이슈·배포 정책은 저장소와 도구로 관리하는 것을 전제로 합니다.

## 폴더 역할

| 경로 | 용도 |
|------|------|
| `E:\01_Master_Keys` | **모든 키·패스키·비밀번호·토큰의 단일 볼트.** 프로젝트에는 값을 복제하지 말고 경로·로더만 둡니다. 자세한 것은 그쪽 `README.md`. |
| `projects/` | **신규** 저장소·클론·다른 앱을 모두 여기에 둡니다. |
| (루트) | 기존에 루트에 있던 프로젝트는 그대로 두어도 됩니다. 점진적으로 `projects/`로 옮길 수 있습니다. |
| `tools/` | 여러 프로젝트에서 쓰는 스크립트·유틸만 둡니다. |
| `.tmp/` | 임시 파일. Git에 올리지 마세요. |

## 한 번에 폴더 만들기

PowerShell에서 워크스페이스 루트로 이동한 뒤:

```powershell
Set-Location 'e:\02_Workspace'
.\setup-workspace.ps1
```

## Cursor에서 안전하게 쓰기

- `.cursor/rules/workspace-isolation.mdc`가 켜져 있으면 Agent가 **워크스페이스 밖 쓰기·시스템 전역 변경**을 기본적으로 피하고, **`E:\01_Master_Keys` 단일 볼트** 원칙을 따릅니다.
- 앱 쪽 `.env.local`에는 **볼트 경로·이름만**, 실제 비밀 값은 볼트에만. 저장소에는 **예시만** (`env.example`).
- 볼트 폴더만 열었을 때는 `E:\01_Master_Keys\.cursor\rules\master-vault.mdc`가 추가로 적용됩니다.

## 터미널·에러 설명 (한·영 + 복붙)

- **뜻**은 짧은 한국어(예: 없다→설치 / 하지 마라), **실제로 칠 명령**은 터미널과 **동일한 문자열**을 **코드 블록에만** 넣어 **복붙→Enter**로 되게 합니다. 블록 안에 `PS>` 같은 프롬프트는 넣지 않습니다.
- 안전하면 Agent가 **먼저 돌려 보고** 확인합니다.
- 규칙 파일: `.cursor/rules/terminal-communication.mdc`

## 더 강한 격리(선택)

| 방식 | 설명 |
|------|------|
| **WSL2** | Linux 전용 빌드·CLI를 게스트에서 실행해 Windows 본체와 분리합니다. |
| **Dev Containers** | VS Code/Cursor의 devcontainer로 프로젝트별 런타임을 컨테이너로 고정합니다. |
| **별도 Windows 사용자** | 동일 PC에서 “실행 전용” 로컬 계정을 두고 권한을 제한할 수 있습니다. |

완전 물리 격리가 필요하면 별도 VM/클라우드 개발 환경을 검토하세요.

## Git

- 루트(`02_Workspace`)가 이미 한 앱의 저장소라면, `projects/` 아래 각 폴더는 **각자 독립 저장소**로 두는 것이 일반적입니다(모노레포가 아닌 한).
