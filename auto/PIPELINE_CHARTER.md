# llangkka · taeja-auto 파이프라인 강령

이 파일만 고치면 지침이 바뀝니다. 코드를 건드릴 필요 없습니다.

**운영(A) vs 제품·SEO(B) 분리 계약**은 저장소 루트 [`docs/PIPELINE_SPLIT.md`](../docs/PIPELINE_SPLIT.md) 를 따른다.

## 1. 숨김·안전 (hide & safety)

- 운영 신호·강령 본문은 **가능하면 외부로 보내지 않는다.** 클라우드 AI를 쓸 때는 `.env`에서 `AUTO_PIPELINE_LOCAL_ONLY=1` 또는 브레인 호출을 끈다.
- 시크릿·개인키·토큰은 감사 로그에 넣지 않는다. API 키는 환경 변수만.
- 외장 드라이브·로컬 워크스페이스에 두는 프로젝트는 **그 경계 안에서** 백업·버전 관리 정책을 따른다.

## 2. 로컬 우선 (local-first)

- 기본 운영 모드는 **로컬에서 크론·봇을 돌리거나**, 배포 시에도 **민감 데이터는 저장하지 않는다.**
- “두뇌”가 필요하면 나중에 **로컬 LLM(Ollama 등)** 을 붙일 수 있게, 판단 로직은 규칙 엔진 + 선택적 모델로 둔다.

## 3. 데이터 경계

- 파이프라인이 수집하는 것은 **헬스 HEAD·도메인 풀·감사 메타** 정도로 제한한다.
- 사용자 생성 콘텐츠 원문은 이 앱에 저장하지 않는다(별도 태자 월드 DB 규칙).

## 4. 확장 — SEO·도구 (llangkka tools)

- SEO 메타 초안·사이트맵 힌트·canonical 검사 등은 **`src/lib/tools/`** 레지스트리에 도구 ID로 등록한다.
- 태자 월드 쪽에서는 같은 모노레포에서 `taeja-auto`의 도구 타입을 import 해 재사용할 수 있다.
- 새 도구 추가 절차: `registry`에 등록 + 이 강령에 한 줄씩 목적을 적는다.

### 등록 예정 도구 ID (채워 넣기)

| ID | 용도 |
|----|------|
| `seo.metaDraft` | 경로·브랜드 규칙으로 title/description 초안 |
| `seo.canonicalCheck` | canonical·호스트 불일치 힌트 |
| `ops.charterExcerpt` | 강령 일부 요약(로컬만) |

## 5. 개정

- 중요한 변경은 커밋 메시지에 `charter:` 접두를 붙인다.
